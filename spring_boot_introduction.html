📚 스프링 부트(Spring Boot) 소개

스프링 부트의 기본 개념과 핵심 기능을 소개합니다.

1. 스프링 부트란?

스프링 부트(Spring Boot)는 스프링 프레임워크를 기반으로 한 애플리케이션을 빠르고 쉽게 개발할 수 있도록 도와주는 도구입니다. 복잡한 설정을 자동화하고, 내장 서버를 제공하여 개발 생산성을 크게 향상시킵니다.

📌 스프링 부트의 주요 특징

  • 자동 구성(Auto-Configuration): 필요한 빈을 자동으로 등록
  • 독립 실행형(Standalone): 내장 톰캣, 제티, 언더토우 등의 서버 포함
  • 의존성 관리: 스타터 의존성을 통한 간편한 의존성 관리
  • 운영 모니터링: 액추에이터를 통한 애플리케이션 모니터링

2. 스프링 부트의 장점

2.1 빠른 개발 시작

Spring Initializr(start.spring.io)를 통해 몇 분 만에 프로젝트를 생성하고 실행할 수 있습니다.

2.2 의존성 관리의 용이성

스프링 부트 스타터 의존성은 관련 라이브러리들을 자동으로 관리해줍니다.

<!-- 예: 웹 애플리케이션 개발을 위한 스타터 -->
<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-web</artifactId>
</dependency>

2.3 내장 서버

별도의 웹 서버 설치 없이도 내장 톰캣, 제티, 언더토우 등을 사용할 수 있습니다.

2.4 프로덕션 준비 기능

액추에이터를 통해 애플리케이션 상태, 메트릭, 상태 점검 등을 모니터링할 수 있습니다.

3. 스프링 부트 애플리케이션 구조

src/main/java/
  └── com/example/demo/
      ├── DemoApplication.java     # 메인 애플리케이션 클래스
      ├── controller/             # 컨트롤러 클래스
      ├── service/                # 서비스 레이어
      ├── repository/             # 데이터 접근 레이어
      └── model/                  # 도메인 모델
src/main/resources/
  ├── static/                 # 정적 리소스 (css, js, 이미지)
  ├── templates/               # 템플릿 파일 (Thymeleaf, Mustache 등)
  ├── application.properties    # 애플리케이션 설정
  └── application-{profile}.properties  # 프로필별 설정

💡 팁: 기본 애플리케이션 클래스

스프링 부트 애플리케이션의 진입점은 @SpringBootApplication 어노테이션이 붙은 메인 클래스입니다.

@SpringBootApplication
public class DemoApplication {
    public static void main(String[] args) {
        SpringApplication.run(DemoApplication.class, args);
    }
}

4. 간단한 REST API 예제

스프링 부트로 간단한 REST API를 만들어보겠습니다.

4.1 컨트롤러 생성

@RestController
@RequestMapping("/api")
public class HelloController {
    
    @GetMapping("/hello")
    public String sayHello(@RequestParam(defaultValue = "World") String name) {
        return String.format("Hello, %s!", name);
    }
    
    @GetMapping("/greeting")
    public Greeting getGreeting(@RequestParam(defaultValue = "World") String name) {
        return new Greeting(1, String.format("Hello, %s!", name));
    }
}

// 간단한 DTO 클래스
class Greeting {
    private final long id;
    private final String content;
    
    // 생성자, getter 생략
}

4.2 애플리케이션 실행

메인 클래스를 실행하면 내장 톰캣 서버가 시작됩니다. 다음 엔드포인트로 접속해보세요:

  • http://localhost:8080/api/hello
  • http://localhost:8080/api/hello?name=Spring
  • http://localhost:8080/api/greeting
