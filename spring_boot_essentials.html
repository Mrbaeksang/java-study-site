📚 스프링 부트 핵심 개념

스프링 부트의 핵심 개념들을 이해하고 효과적으로 활용하는 방법을 배웁니다.

1. 자동 구성(Auto-Configuration)

스프링 부트의 핵심 기능 중 하나로, 클래스패스에 있는 라이브러리들을 기반으로 자동으로 스프링 애플리케이션을 구성해줍니다.

주요 특징

  • 의존성 기반 자동 구성
  • 조건부 빈 등록
  • 사용자 정의 구성 오버라이딩
@SpringBootApplication
public class MyApplication {
    public static void main(String[] args) {
        SpringApplication.run(MyApplication.class, args);
    }
}

2. 스타터 의존성(Starter Dependencies)

공통적인 기능을 쉽게 추가할 수 있도록 미리 구성된 의존성 그룹입니다.

주요 스타터

  • spring-boot-starter-web: 웹 애플리케이션 개발
  • spring-boot-starter-data-jpa: JPA를 사용한 데이터 접근
  • spring-boot-starter-security: 보안 기능
  • spring-boot-starter-test: 테스트 지원

3. 외부 구성(Externalized Configuration)

애플리케이션 설정을 외부에서 관리할 수 있는 다양한 방법을 제공합니다.

주요 설정 방식

  • application.properties 또는 application.yml 파일
  • 환경 변수
  • 명령줄 인수
  • 프로필별 설정
# application.yml 예시
server:
  port: 8080
  servlet:
    context-path: /api

spring:
  datasource:
    url: jdbc:h2:mem:testdb
    username: sa
    password: password
  jpa:
    hibernate:
      ddl-auto: update
    show-sql: true

4. 내장 서버(Embedded Server)

톰캣(Tomcat), 제티(Jetty), 언더토우(Undertow)와 같은 서블릿 컨테이너를 내장하고 있어 별도의 웹 서버 설치 없이 실행할 수 있습니다.

💡 팁: spring-boot-starter-web을 추가하면 기본적으로 톰캣이 내장 서버로 사용됩니다.

5. 액추에이터(Actuator)

애플리케이션의 모니터링과 관리를 위한 다양한 엔드포인트를 제공합니다.

주요 엔드포인트

  • /actuator/health: 애플리케이션 상태 확인
  • /actuator/info: 애플리케이션 정보
  • /actuator/metrics: 메트릭 정보
  • /actuator/env: 환경 속성
# 의존성 추가
implementation 'org.springframework.boot:spring-boot-starter-actuator'

# application.yml 설정
management:
  endpoints:
    web:
      exposure:
        include: "*"
  endpoint:
    health:
      show-details: always
