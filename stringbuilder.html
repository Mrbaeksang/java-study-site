📚 StringBuilder 개요

StringBuilder는 가변(mutable) 문자열을 효율적으로 처리하기 위한 클래스입니다.

주요 특징

  • 문자열 수정 가능 (가변)
  • 스레드 안전하지 않음 (단일 스레드에서 사용)
  • 문자열 연산 성능 최적화
  • 메모리 효율성

String vs StringBuilder

특성 String StringBuilder
가변성 불변(Immutable) 가변(Mutable)
스레드 안전성 안전 불안전
성능 수정 시 새 객체 생성 수정 시 기존 객체 재사용
메모리 수정마다 새로운 메모리 동적으로 크기 조절

🛠 기본 메서드

append()

추가
자주 사용
append(String str)

문자열 끝에 새로운 내용을 추가합니다.

StringBuilder sb = new StringBuilder("Hello");
sb.append(" World"); // "Hello World"

insert()

삽입
insert(int offset, String str)

지정된 위치에 문자열을 삽입합니다.

StringBuilder sb = new StringBuilder("Hello World");
sb.insert(5, " Beautiful"); // "Hello Beautiful World"

delete()

삭제
delete(int start, int end)

지정된 범위의 문자를 삭제합니다.

StringBuilder sb = new StringBuilder("Hello World");
sb.delete(5, 11); // "Hello"

✏️ 수정 메서드

replace()

수정
replace(int start, int end, String str)

지정된 범위의 문자열을 새로운 문자열로 교체합니다.

StringBuilder sb = new StringBuilder("Hello World");
sb.replace(6, 11, "Java"); // "Hello Java"

reverse()

변환
reverse()

문자열을 역순으로 뒤집습니다.

StringBuilder sb = new StringBuilder("Hello");
sb.reverse(); // "olleH"

💡 실전 예제

1. CSV 문자열 생성

public static String createCSV(String... items) { StringBuilder sb = new StringBuilder(); for (int i = 0; i < items.length; i++) { sb.append(items[i]); if (i < items.length - 1) { sb.append(","); } } return sb.toString(); } // 사용 예 String csv = createCSV("apple", "banana", "orange"); // "apple,banana,orange"

2. 문자열 반복

public static String repeat(String str, int times) { StringBuilder sb = new StringBuilder(); for (int i = 0; i < times; i++) { sb.append(str); } return sb.toString(); } // 사용 예 String repeated = repeat("Ha", 3); // "HaHaHa"

3. URL 파라미터 생성

public static String buildUrl(String baseUrl, Map params) { StringBuilder sb = new StringBuilder(baseUrl); if (!params.isEmpty()) { sb.append('?'); for (Map.Entry entry : params.entrySet()) { sb.append(entry.getKey()) .append('=') .append(entry.getValue()) .append('&'); } sb.setLength(sb.length() - 1); // 마지막 & 제거 } return sb.toString(); } // 사용 예 Map params = new HashMap<>(); params.put("name", "John"); params.put("age", "30"); String url = buildUrl("http://example.com", params); // "http://example.com?name=John&age=30"

⚡ 성능 최적화 팁

1. 초기 용량 설정

예상되는 최종 문자열 길이로 초기 용량을 설정하면 메모리 재할당을 줄일 수 있습니다.

// 권장 StringBuilder sb = new StringBuilder(1000);
// 비권장 StringBuilder sb = new StringBuilder();

2. 메서드 체이닝 활용

여러 작업을 한 번에 수행할 수 있습니다.

StringBuilder sb = new StringBuilder() .append("Hello") .append(" ") .append("World") .append("!");

주의사항

  • 스레드 안전성이 필요한 경우 StringBuffer 사용
  • 불필요한 toString() 호출 피하기
  • 적절한 초기 용량 설정으로 성능 최적화
