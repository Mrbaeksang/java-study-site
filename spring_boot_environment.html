⚙️ 스프링 부트 개발 환경 설정

스프링 부트 개발을 위한 필수 도구들을 설치하고 설정하는 방법을 안내합니다.

1. 필수 도구 설치

1 JDK 설치

스프링 부트 3.x는 Java 17 이상을 요구합니다. OpenJDK나 Oracle JDK 중 선택하여 설치하세요.

📌 권장 JDK 버전

  • Java 17 (LTS) - 안정적인 버전 추천
  • Java 21 (LTS) - 최신 LTS 버전

설치 확인:

java -version
javac -version

2 빌드 도구 선택

Maven이나 Gradle 중 선택하세요. 최신 스프링 부트 프로젝트는 Gradle을 권장합니다.

Maven (pom.xml)

<parent>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-parent</artifactId>
    <version>3.2.0</version>
</parent>

<dependencies>
    <dependency>
        <groupId>org.springframework.boot</groupId>
        <artifactId>spring-boot-starter-web</artifactId>
    </dependency>
</dependencies>

Gradle (build.gradle)

plugins {
    id 'java'
    id 'org.springframework.boot' version '3.2.0'
    id 'io.spring.dependency-management' version '1.1.4'
}

dependencies {
    implementation 'org.springframework.boot:spring-boot-starter-web'
}

설치 확인:

# Maven 확인
mvn -v

# Gradle 확인
gradle -v

3 IDE 선택 및 설정

다음 IDE 중 하나를 선택하세요. IntelliJ IDEA를 가장 추천합니다.

  • IntelliJ IDEA (Ultimate/Community) - 다운로드
  • Eclipse IDE - 다운로드
  • VS Code - 다운로드 (Extension Pack for Java 설치 필요)

💡 IntelliJ IDEA 플러그인 추천

  • Lombok
  • Spring Boot Tools
  • Spring Initializr and Assistant
  • MapStruct Support

2. 프로젝트 생성하기

1 Spring Initializr 사용하기

start.spring.io에서 프로젝트를 생성하거나, IDE에서 Spring Initializr를 사용하세요.

📌 권장 설정

  • Project: Gradle (Kotlin) or Maven
  • Language: Java
  • Spring Boot: 3.2.x (최신 안정 버전)
  • Packaging: Jar (기본값)
  • Java: 17 or 21

2 의존성 추가

필요한 스타터 의존성을 선택하세요. 예시:

  • Spring Web - 웹 애플리케이션 개발
  • Spring Data JPA - 데이터베이스 연동
  • Lombok - 보일러플레이트 코드 감소
  • Spring Boot DevTools - 개발자 도구
  • Spring Security - 인증/인가

3 프로젝트 가져오기

생성된 프로젝트를 IDE로 가져옵니다.

IntelliJ IDEA

  1. File > Open
  2. 생성된 프로젝트 폴더 선택
  3. Trust Project 선택

VS Code

  1. File > Open Folder
  2. 생성된 프로젝트 폴더 선택
  3. 필요한 확장 프로그램 설치 안내가 표시되면 설치

3. 애플리케이션 실행 및 확인

1 메인 클래스 실행

기본 애플리케이션 클래스(@SpringBootApplication이 붙은 클래스)를 실행하세요.

@SpringBootApplication
public class MyApplication {
    public static void main(String[] args) {
        SpringApplication.run(MyApplication.class, args);
    }
}

2 실행 확인

애플리케이셔니 정상적으로 실행되면 다음 로그가 표시됩니다:

3 테스트 엔드포인트 추가

간단한 REST 컨트롤러를 추가하여 테스트해보세요.

@RestController
public class HelloController {
    
    @GetMapping("/")
    public String hello() {
        return "Hello, Spring Boot!";
    }
}

브라우저에서 http://localhost:8080에 접속하여 확인하세요.

4. 문제 해결

⚠️ 포트 충돌 문제

기본 포트(8080)가 이미 사용 중이라면 application.properties에서 포트를 변경하세요:

server.port=8081

⚠️ 의존성 문제

의존성 문제가 발생하면 다음 명령어로 의존성을 새로 고치세요:

# Maven
mvn clean install

# Gradle
gradle --refresh-dependencies

💡 개발자 도구 (DevTools)

개발 중 자동 재시작과 같은 기능을 사용하려면 spring-boot-devtools 의존성을 추가하세요.

<!-- Maven -->
<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-devtools</artifactId>
    <scope>runtime</scope>
    <optional>true</optional>
</dependency>
