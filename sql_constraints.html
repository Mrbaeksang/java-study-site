🔒 제약조건

데이터 무결성을 보장하는 제약조건의 종류와 사용법을 배워보세요.

1. NOT NULL 제약조건

NOT NULL 제약조건은 열에 NULL 값을 허용하지 않습니다.

📝 예제: NOT NULL 사용

CREATE TABLE students ( student_id INT PRIMARY KEY, name VARCHAR(50) NOT NULL, department VARCHAR(50) NOT NULL );

2. UNIQUE 제약조건

UNIQUE 제약조건은 열에 중복된 값을 허용하지 않습니다.

📝 예제: UNIQUE 사용

CREATE TABLE students ( student_id INT PRIMARY KEY, email VARCHAR(100) UNIQUE );

3. CHECK 제약조건

CHECK 제약조건은 특정 조건을 만족하는 값만 허용합니다.

📝 예제: CHECK 사용

CREATE TABLE students ( student_id INT PRIMARY KEY, grade INT CHECK (grade BETWEEN 1 AND 4) );

4. DEFAULT 제약조건

DEFAULT 제약조건은 INSERT 시 값을 지정하지 않으면 기본값을 사용합니다.

📝 예제: DEFAULT 사용

CREATE TABLE students ( student_id INT PRIMARY KEY, admission_date DATE DEFAULT CURRENT_DATE );

5. PRIMARY KEY 제약조건

PRIMARY KEY 제약조건은 테이블의 행을 고유하게 식별합니다.

📝 예제: PRIMARY KEY 사용

CREATE TABLE students ( student_id INT PRIMARY KEY, name VARCHAR(50) );

6. FOREIGN KEY 제약조건

FOREIGN KEY 제약조건은 다른 테이블의 PRIMARY KEY를 참조합니다.

📝 예제: FOREIGN KEY 사용

CREATE TABLE departments ( dept_id INT PRIMARY KEY, dept_name VARCHAR(50) ); CREATE TABLE students ( student_id INT PRIMARY KEY, dept_id INT, FOREIGN KEY (dept_id) REFERENCES departments(dept_id) );

🎯 제약조건 주의사항

  • 제약조건은 데이터 무결성을 보장하지만 성능에 영향을 줄 수 있습니다.
  • 필요한 제약조건만 사용하는 것이 좋습니다.
  • 제약조건은 테이블 생성 시나 나중에 ALTER TABLE을 사용하여 추가할 수 있습니다.
