🍎 HashSet은 왜 필요할까?

실생활에서 HashSet이 필요한 순간!
  • 중복 없이 유일한 값만 저장하고 싶을 때
  • 빠른 검색과 집합 연산이 필요할 때
// 중복 제거 예시 HashSet set = new HashSet<>(); set.add("A"); set.add("B"); set.add("A"); System.out.println(set); // [A, B]
HashSet은 중복 불가 & 순서 없음의 집합 자료구조입니다!

📖 HashSet 클래스 개요

  • 중복 불가: 동일 값 한 번만 저장
  • 순서 없음: 저장 순서 유지 X
  • 빠른 검색, 추가, 삭제 지원

🔑 주요 메서드 한눈에 보기 & 실전 활용

add()

  • 언제/왜? 새로운 값을 집합에 추가할 때. 이미 있으면 추가 안 됨(중복 불가).
  • 코드 예시:
    HashSet set = new HashSet<>(); System.out.println(set.add("A")); // true System.out.println(set.add("A")); // false (이미 존재) System.out.println(set); // [A]
  • 실전 활용: 중복 제거, for문에서 조건부 add, 집합 연산 등
  • 함께 쓰면 좋은 메서드: contains(), size()
  • 자주 하는 실수: add()가 false 반환 → 이미 들어있는 값
미션: "A", "B", "A"를 add한 후 set의 크기는?
정답 보기
2 (중복 불가)

remove()

  • 언제/왜? 집합에서 특정 값을 삭제할 때
  • 코드 예시:
    HashSet set = new HashSet<>(); set.add("A"); set.add("B"); System.out.println(set.remove("B")); // true System.out.println(set.remove("C")); // false System.out.println(set); // [A]
  • 실전 활용: 조건부 삭제, 반복문 내 안전 삭제(Iterator와 조합)
  • 자주 하는 실수: 없는 값 삭제 시 false 반환
미션: "A", "B"를 add 후 "B"를 remove하면 set에 남은 값은?
정답 보기
[A]

contains()

  • 언제/왜? 값이 집합에 존재하는지 확인할 때
  • 코드 예시:
    HashSet set = new HashSet<>(); set.add("A"); System.out.println(set.contains("A")); // true System.out.println(set.contains("B")); // false
  • 실전 활용: 중복 체크, 조건부 처리, if문과 조합
미션: set에 "A"만 있을 때 set.contains("B") 결과는?
정답 보기
false

size()

  • 언제/왜? 집합에 저장된 값의 개수 확인
  • 코드 예시:
    HashSet set = new HashSet<>(); set.add(1); set.add(2); System.out.println(set.size()); // 2
  • 실전 활용: 입력값 개수 제한, 집합 연산 결과 확인
미션: set에 1, 2, 2를 add하면 size()는?
정답 보기
2

clear()

  • 언제/왜? 집합의 모든 값을 한 번에 삭제할 때
  • 코드 예시:
    HashSet set = new HashSet<>(); set.add("A"); set.add("B"); set.clear(); System.out.println(set); // []
  • 실전 활용: 상태 초기화, 재사용
미션: clear() 후 set.isEmpty() 결과는?
정답 보기
true

isEmpty()

  • 언제/왜? 집합이 비었는지 확인할 때
  • 코드 예시:
    HashSet set = new HashSet<>(); System.out.println(set.isEmpty()); // true set.add("A"); System.out.println(set.isEmpty()); // false
  • 실전 활용: 반복문 조건, 입력/출력 제어
미션: set에 아무 값도 없으면 isEmpty()는?
정답 보기
true

❓ 자주 하는 질문 (FAQ)

Q. HashSet과 ArrayList의 차이?
HashSet은 중복 불가, ArrayList는 중복 허용!
Q. 순서가 중요한 경우엔?
LinkedHashSet, TreeSet 등 다른 Set 사용!
Q. null 값도 저장 가능한가요?
네! 단, 한 번만 저장 가능

⚡ 성능과 주의사항

성능 특성
  • 검색, 추가, 삭제가 매우 빠름 (평균 O(1))
  • 순서 유지가 필요하면 HashSet 대신 LinkedHashSet 사용
실무 사용 팁
  • 중복 제거, 빠른 검색, 집합 연산에 활용
  • Set 인터페이스로 선언하면 TreeSet 등으로 교체 용이
