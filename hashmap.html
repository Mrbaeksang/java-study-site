📚 HashMap 개요

HashMap은 키-값 쌍을 저장하는 자료구조로, 빠른 데이터 검색이 필요할 때 사용됩니다.

주요 특징

  • 키-값 쌍으로 데이터 저장
  • O(1) 시간 복잡도로 검색
  • 키는 중복될 수 없음
  • 값은 중복 가능
  • null 키와 값 허용

HashMap vs HashTable vs TreeMap

특성 HashMap HashTable TreeMap
동기화 비동기 동기화 비동기
Null 허용 키/값 모두 불가능 값만
순서 무작위 무작위 정렬됨
성능 빠름 보통 느림

🛠 기본 메서드

put()

추가/수정
자주 사용
put(K key, V value)

키-값 쌍을 추가하거나 수정합니다.

HashMap map = new HashMap<>(); map.put("Apple", 1); map.put("Banana", 2);

get()

조회
자주 사용
get(Object key)

키에 해당하는 값을 반환합니다.

Integer value = map.get("Apple"); // 1

remove()

삭제
remove(Object key)

키에 해당하는 항목을 삭제합니다.

map.remove("Apple"); // 제거

🔍 검색과 확인

containsKey()

검색
containsKey(Object key)

키의 존재 여부를 확인합니다.

boolean hasKey = map.containsKey("Apple");

containsValue()

검색
containsValue(Object value)

값의 존재 여부를 확인합니다.

boolean hasValue = map.containsValue(1);

getOrDefault()

조회
안전
getOrDefault(Object key, V defaultValue)

키가 없을 때 기본값을 반환합니다.

int value = map.getOrDefault("Orange", 0);

💡 실전 예제

1. 단어 빈도수 계산

String text = "apple banana apple cherry banana apple"; HashMap frequency = new HashMap<>(); for (String word : text.split(" ")) { frequency.put(word, frequency.getOrDefault(word, 0) + 1); } // 결과: {apple=3, banana=2, cherry=1}

2. 그룹화

ArrayList fruits = new ArrayList<>(Arrays.asList( "Apple", "Banana", "Cherry", "Date", "Elderberry" )); HashMap> groups = new HashMap<>(); for (String fruit : fruits) { char firstChar = fruit.charAt(0); groups.computeIfAbsent(firstChar, k -> new ArrayList<>()).add(fruit); } // 결과: {A=[Apple], B=[Banana], C=[Cherry], D=[Date], E=[Elderberry]}

3. 캐시 구현

class SimpleCache { private final HashMap cache = new HashMap<>(); private final int capacity; public SimpleCache(int capacity) { this.capacity = capacity; } public V get(K key) { return cache.get(key); } public void put(K key, V value) { if (cache.size() >= capacity) { K firstKey = cache.keySet().iterator().next(); cache.remove(firstKey); } cache.put(key, value); } }

⚡ 성능 최적화 팁

1. 초기 용량 설정

예상되는 크기로 초기 용량을 설정하면 재해시를 줄일 수 있습니다.

// 권장 HashMap map = new HashMap<>(1000);
// 비권장 HashMap map = new HashMap<>();

2. equals()와 hashCode() 구현

커스텀 키 클래스는 반드시 두 메서드를 올바르게 구현해야 합니다.

class CustomKey { private final String id; @Override public boolean equals(Object o) { if (this == o) return true; if (!(o instanceof CustomKey)) return false; CustomKey that = (CustomKey) o; return Objects.equals(id, that.id); } @Override public int hashCode() { return Objects.hash(id); } }

주의사항

  • 키의 불변성 유지
  • 적절한 초기 용량 설정
  • 반복문에서의 효율적인 접근
