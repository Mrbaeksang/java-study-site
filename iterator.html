🍎 Iterator는 왜 필요할까?

실생활에서 Iterator가 필요한 순간!
  • 리스트, Set 등 모든 컬렉션을 반복문 없이 순회하고 싶을 때
  • for-each문으로 처리할 수 없는 삭제/수정이 필요할 때
// Iterator로 요소 출력 Iterator it = list.iterator(); while(it.hasNext()) { System.out.println(it.next()); }
Iterator는 컬렉션 순회를 표준화한 반복자입니다!

📖 Iterator 클래스 개요

  • 모든 컬렉션에서 iterator()로 획득
  • hasNext(), next(), remove() 등 메서드 제공
  • for-each문 내부적으로도 Iterator 사용

🔑 주요 메서드 한눈에 보기 & 실전 활용

hasNext()

  • 언제/왜? 다음 요소가 남아있는지 확인할 때
  • 코드 예시:
    Iterator it = list.iterator(); while(it.hasNext()) { System.out.println(it.next()); }
  • 실전 활용: 반복문 종료 조건, 안전한 순회
미션: hasNext()가 false면 반복문은?
정답 보기
종료

next()

  • 언제/왜? 다음 요소를 꺼낼 때
  • 코드 예시:
    Iterator it = list.iterator(); if(it.hasNext()) { String s = it.next(); System.out.println(s); }
  • 실전 활용: 요소 접근, 순회, 조건부 처리
  • 자주 하는 실수: hasNext() 없이 next() 호출 시 예외
미션: hasNext()가 false인데 next() 호출하면?

예외 발생(NoSuchElementException)

remove()

  • 언제/왜? 현재 가리키는 요소를 안전하게 삭제할 때
  • 코드 예시:
    Iterator it = list.iterator(); while(it.hasNext()) { String s = it.next(); if(s.equals("A")) it.remove(); }
  • 실전 활용: 반복문 내 안전 삭제, ConcurrentModificationException 방지
  • 자주 하는 실수: next() 없이 remove() 호출 시 예외

❓ 자주 하는 질문 (FAQ)

Q. for-each와 Iterator 차이?
for-each는 내부적으로 Iterator 사용, 삭제는 Iterator만 가능!
Q. remove()는 언제 사용하나요?
반복 중 안전하게 삭제할 때 사용
Q. 여러 컬렉션에서 공통 사용 가능한가요?
네! List, Set, Map 등 모든 컬렉션에서 사용

⚡ 성능과 주의사항

성능 특성
  • for문보다 느릴 수 있으나, 안전한 삭제/수정에 필수
  • remove()는 next() 호출 후 1회만 가능
실무 사용 팁
  • for-each로 순회하다가 삭제 필요하면 Iterator로 변경
  • ConcurrentModificationException 주의(반복 중 컬렉션 직접 수정 X)
