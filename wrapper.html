🍎 Wrapper 클래스는 왜 필요할까?

실생활과 개발에서 Wrapper가 필요한 순간!
  • 컬렉션에 int, double 등 기본형을 담고 싶을 때
  • 문자열 ↔ 숫자 변환, null 처리, 비교 등 객체 기능이 필요할 때
  • 제네릭, Stream, Optional 등 객체만 지원하는 API 사용 시
Wrapper 클래스는 기본 타입을 객체로 감싸 컬렉션, 변환, 비교 등에 활용할 수 있게 해줍니다!

📖 Wrapper 클래스 개요

  • 기본형(primitive) → 객체(Object)로 변환: 박싱(Boxing)
  • 객체(Object) → 기본형(primitive)으로 변환: 언박싱(Unboxing)
  • 모든 Wrapper는 불변(immutable)
  • 상수, 변환, 비교 등 다양한 메서드 제공

❓ 자주 하는 질문 (FAQ)

Q. Wrapper 클래스는 언제 꼭 필요할까?
컬렉션, 제네릭, null 처리, 문자열 변환 등 객체로서 기능이 필요할 때!
Q. ==와 equals의 차이?
==는 참조비교, equals는 값비교!
Q. 박싱/언박싱은 언제 자동?
대입, 연산 등에서 컴파일러가 자동 변환해줍니다.
Q. null과 비교할 때 주의점?
equals() 호출 전 반드시 null 체크!
Q. Integer.valueOf와 new Integer의 차이?
valueOf는 캐싱(재사용), new는 무조건 새 객체 생성(비추천)

⚡ 성능과 주의사항

성능 특성
  • == 연산자 대신 equals() 사용
  • 불필요한 객체 생성 피하기 (valueOf/parse 계열 활용)
  • null 체크 필수
  • Wrapper는 불변(immutable)이라 값 변경 불가
  • 박싱/언박싱 반복 시 성능 저하 가능
실무 사용 팁
  • 상수(Integer.MAX_VALUE 등) 적극 활용
  • 문자열 변환/파싱 시 NumberFormatException 주의
  • Optional 등과 함께 null 안전하게 활용
