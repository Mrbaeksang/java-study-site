1️⃣ 관계형 DB 구조 이해

테이블, 행, 열, 키 개념을 체계적으로 학습합니다.

1. 관계형 데이터베이스 기본 개념

관계형 데이터베이스는 데이터를 표 형태의 테이블로 저장하고 관리하는 시스템입니다. 각 테이블은 관련된 데이터를 그룹화하여 저장합니다.

📝 예제: 학생 정보 테이블

학번 이름 학과 학년
20230001 김철수 컴퓨터공학 2
20230002 이영희 전자공학 3

2. 테이블 구조 이해

테이블은 행과 열로 구성됩니다:

  • 행(Record/Row): 테이블의 한 줄을 의미합니다.
  • 열(Column/Field): 테이블의 한 칸을 의미합니다.

3. 키(Key)의 종류

📝 키의 종류

  • 프라이머리 키(Primary Key): 테이블의 각 행을 고유하게 식별하는 키
  • 포리너 키(Foreign Key): 다른 테이블의 프라이머리 키를 참조하는 키
  • 유니크 키(Unique Key): 중복되지 않는 값을 가진 키

4. 데이터 관계 이해

테이블 간의 관계는 다음과 같이 나눌 수 있습니다:

  • 1:1 관계 (일대일)
  • 1:N 관계 (일대다)
  • N:M 관계 (다대다)

💡 팁: 데이터 모델링의 중요성

데이터베이스 설계 시 적절한 테이블 구조와 관계를 설계하는 것이 성능과 유지보수에 중요합니다.
