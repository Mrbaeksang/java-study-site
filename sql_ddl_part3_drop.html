🗑️ DROP 명령어

테이블, 인덱스, 뷰 등을 삭제하는 방법을 배워보세요.

1. DROP TABLE

DROP TABLE 명령어는 테이블을 완전히 삭제합니다.

📝 예제: 테이블 삭제

DROP TABLE students;

⚠️ 주의사항

DROP TABLE은 테이블과 그 안의 모든 데이터를 완전히 삭제합니다. 복구가 불가능하므로 주의가 필요합니다.

2. CASCADE 옵션

CASCADE 옵션을 사용하면 참조하는 테이블들도 함께 삭제됩니다.

📝 예제: CASCADE 옵션 사용

DROP TABLE students CASCADE;

🎯 CASCADE 옵션 주의사항

  • CASCADE 옵션은 데이터 무결성을 유지하기 위해 사용됩니다.
  • 하지만 주의가 필요하며, 가능하면 참조 무결성 제약조건을 먼저 제거하는 것이 좋습니다.

3. TRUNCATE TABLE

TRUNCATE TABLE은 테이블의 데이터만 삭제하고 테이블 구조는 유지합니다.

📝 예제: TRUNCATE 사용

TRUNCATE TABLE students;

🎯 TRUNCATE와 DELETE의 차이점

  • TRUNCATE: 데이터만 삭제, 빠르고 롤백 불가능
  • DELETE: 데이터 삭제 시 트랜잭션 관리 가능
