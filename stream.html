📚 Stream 개요

Stream API는 컬렉션의 요소를 함수형 스타일로 처리할 수 있게 해주는 현대적인 방식입니다.

주요 특징

  • 파이프라인 형태의 연산
  • 중간 연산과 최종 연산
  • 지연 평가 (Lazy Evaluation)
  • 내부 반복
  • 병렬 처리 지원
컬렉션
→
필터링
→
변환
→
수집

Stream vs Collection

특성 Stream Collection
데이터 접근 한 번만 여러 번
처리 방식 함수형 반복문
처리 시점 지연 처리 즉시 처리
병렬 처리 쉬움 어려움

🔄 중간 연산 (Intermediate Operations)

filter()

필터링
자주 사용
filter(Predicate predicate)

조건에 맞는 요소만 선택합니다.

numbers.stream() .filter(n -> n > 0) .collect(Collectors.toList());

map()

변환
자주 사용
map(Function mapper)

각 요소를 변환합니다.

strings.stream() .map(String::toUpperCase) .collect(Collectors.toList());

sorted()

정렬
sorted()

요소를 정렬합니다.

numbers.stream() .sorted() .collect(Collectors.toList());

🎯 최종 연산 (Terminal Operations)

collect()

수집
자주 사용
collect(Collector collector)

결과를 컬렉션으로 변환합니다.

stream.collect(Collectors.toList()); stream.collect(Collectors.toSet()); stream.collect(Collectors.toMap(k -> k, v -> v));

reduce()

집계
reduce(BinaryOperator accumulator)

요소들을 하나로 줄입니다.

numbers.stream() .reduce(0, (a, b) -> a + b);

forEach()

순회
forEach(Consumer action)

각 요소에 대해 작업을 수행합니다.

stream.forEach(System.out::println);

💡 실전 예제

1. 데이터 필터링과 변환

List result = people.stream() .filter(p -> p.getAge() > 20) .map(Person::getName) .collect(Collectors.toList());

2. 그룹화와 집계

Map avgAgeByDept = employees.stream() .collect(Collectors.groupingBy( Employee::getDepartment, Collectors.averagingInt(Employee::getAge) ));

3. 병렬 처리

long count = numbers.parallelStream() .filter(n -> n % 2 == 0) .map(n -> n * 2) .count();

⚡ 성능 최적화 팁

1. 적절한 병렬 처리

데이터가 충분히 많을 때만 병렬 스트림을 사용하세요.

// 데이터가 많을 때 hugeList.parallelStream()... // 데이터가 적을 때 smallList.stream()...

2. 효율적인 연산 순서

필터링을 먼저하고 변환하세요.

// 좋음 stream.filter(...).map(...) // 나쁨 stream.map(...).filter(...)

주의사항

  • 스트림은 한 번만 사용 가능
  • 상태 변경 연산 주의
  • 무한 스트림 사용 시 주의
  • 병렬 스트림의 부작용 고려
