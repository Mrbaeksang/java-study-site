📚 예외 처리 개요

예외 처리는 프로그램 실행 중 발생할 수 있는 오류를 안전하게 처리하는 메커니즘입니다.

예외의 종류

  • Throwable
    • Error (시스템 에러)
      • OutOfMemoryError
      • StackOverflowError
    • Exception (프로그램 에러)
      • RuntimeException (Unchecked)
        • NullPointerException
        • ArrayIndexOutOfBoundsException
        • ClassCastException
      • IOException (Checked)
        • FileNotFoundException
        • EOFException

Checked vs Unchecked Exceptions

특성 Checked Exception Unchecked Exception
처리 필수 예 (컴파일 에러) 아니오
상속 계층 Exception RuntimeException
대표 예시 IOException, SQLException NullPointerException, IndexOutOfBoundsException
발생 원인 외부 요인 프로그래밍 오류

🛠 기본 문법

try-catch

필수
기본
try { // 예외 발생 가능 코드 } catch (Exception e) { // 예외 처리 코드 }

기본적인 예외 처리 구문입니다.

try-catch-finally

리소스 정리
try { // 예외 발생 가능 코드 } catch (Exception e) { // 예외 처리 코드 } finally { // 항상 실행되는 코드 }

리소스 정리가 필요할 때 사용합니다.

try-with-resources

모던 자바
자동 정리
try (Resource res = new Resource()) { // 리소스 사용 코드 } catch (Exception e) { // 예외 처리 코드 }

자동으로 리소스를 닫아줍니다.

💡 실전 예제

1. 다중 예외 처리

try { int[] arr = new int[5]; arr[10] = 50; // ArrayIndexOutOfBoundsException int result = 10 / 0; // ArithmeticException } catch (ArrayIndexOutOfBoundsException e) { System.out.println("배열 인덱스 오류: " + e.getMessage()); } catch (ArithmeticException e) { System.out.println("산술 오류: " + e.getMessage()); } catch (Exception e) { System.out.println("기타 오류: " + e.getMessage()); }

2. 파일 처리 예외

try (FileReader fr = new FileReader("file.txt"); BufferedReader br = new BufferedReader(fr)) { String line; while ((line = br.readLine()) != null) { System.out.println(line); } } catch (IOException e) { System.err.println("파일 처리 오류: " + e.getMessage()); }

3. 사용자 정의 예외

class InvalidAgeException extends Exception { public InvalidAgeException(String message) { super(message); } } public void checkAge(int age) throws InvalidAgeException { if (age < 0) { throw new InvalidAgeException("나이는 음수일 수 없습니다."); } }

⚡ 성능과 모범 사례

1. 예외는 정말 예외적인 상황에만

일반적인 흐름 제어에 예외를 사용하지 마세요.

// 나쁜 예 try { while (true) { process(); } } catch (EndOfProcessException e) { } // 좋은 예 while (hasMoreToProcess()) { process(); }

2. 구체적인 예외 처리

가능한 구체적인 예외를 먼저 잡으세요.

try { // 코드 } catch (FileNotFoundException e) { // 구체적인 처리 } catch (IOException e) { // 더 일반적인 처리 } catch (Exception e) { // 가장 일반적인 처리 }

주의사항

  • 빈 catch 블록 피하기
  • 예외 처리 시 로깅 필수
  • finally 블록에서 예외 발생 주의
  • 예외 전환 시 원인 유지
