📝 SQL DDL (Data Definition Language)

데이터베이스 스키마를 정의하고 관리하는 DDL(Data Definition Language)에 대해 학습합니다. 테이블 생성, 수정, 삭제 등 데이터베이스 객체를 관리하는 방법을 알아봅니다.

🔍 DDL 개요

DDL(Data Definition Language)은 데이터베이스의 구조를 정의하고 변경하는 데 사용되는 SQL의 하위 언어입니다. 주로 데이터베이스 객체(테이블, 인덱스, 뷰 등)를 생성, 수정, 삭제하는 데 사용됩니다.

📌 주요 DDL 명령어

  • CREATE - 데이터베이스 객체 생성
  • ALTER - 기존 데이터베이스 객체 수정
  • DROP - 데이터베이스 객체 삭제
  • TRUNCATE - 테이블의 모든 행 삭제 (구조는 유지)
  • RENAME - 데이터베이스 객체 이름 변경
  • COMMENT - 데이터베이스 객체에 주석 추가

📌 DDL 모범 사례

💡 DDL 사용 시 주의사항

  • 백업 필수: DDL 명령은 데이터베이스 구조를 변경하므로 실행 전 반드시 백업을 수행하세요.
  • 트랜잭션 주의: 일부 데이터베이스에서는 DDL 명령이 자동 커밋되므로 주의가 필요합니다.
  • 성능 고려: 대용량 테이블의 ALTER 작업은 시스템 부하를 유발할 수 있습니다.
  • 의존성 확인: 객체를 삭제하거나 수정할 때는 해당 객체를 참조하는 다른 객체가 없는지 확인하세요.
