📚 SQL 소개

SQL(Structured Query Language)의 기본 개념과 역사에 대해 알아봅시다.

1. SQL이란 무엇인가요?

SQL(Structured Query Language)은 관계형 데이터베이스 관리 시스템(RDBMS)에서 데이터를 관리하고 조작하기 위해 사용되는 표준 프로그래밍 언어입니다.

💡 핵심 특징

  • 데이터베이스와 상호작용하기 위한 표준 언어
  • 데이터 정의, 조작, 제어를 위한 다양한 명령어 제공
  • 선언적 언어로 '무엇을' 할지에 초점을 맞춤
  • ANSI/ISO 표준을 따르지만, 각 DBMS마다 약간의 차이가 있음

2. SQL의 역사

SQL은 1970년대 초 IBM 연구소에서 개발된 SEQUEL(Structured English Query Language)에서 시작되었습니다.

주요 연혁

연도 사건
1970년 E.F. Codd가 관계형 데이터베이스 모델 제안
1974년 IBM에서 SEQUEL 개발 시작
1979년 Oracle에서 최초의 상용 RDBMS 출시
1986년 ANSI SQL 표준 채택 (SQL-86)
1992년 SQL-92 (SQL2) 표준화
1999년 SQL:1999 (SQL3) - 재귀 쿼리, 트리거 등 추가
2003년~현재 XML 지원, 윈도우 함수, JSON 지원 등 지속적인 확장

3. RDBMS 이해하기

관계형 데이터베이스 관리 시스템(RDBMS)은 테이블 형태로 데이터를 저장하고, 이들 간의 관계를 정의하여 데이터를 관리하는 시스템입니다.

주요 RDBMS 제품

  • MySQL: 오픈 소스 RDBMS로 웹 애플리케이션에 널리 사용
  • Oracle Database: 기업용 고성능 데이터베이스
  • Microsoft SQL Server: 마이크로소프트의 통합 데이터 플랫폼
  • PostgreSQL: 고급 기능을 갖춘 오픈 소스 RDBMS
  • SQLite: 임베디드 데이터베이스로 가볍고 사용이 간편

📌 RDBMS의 주요 특징

  • 데이터는 테이블 형태로 구성됨
  • 테이블 간의 관계를 통해 데이터 중복 최소화
  • ACID(원자성, 일관성, 고립성, 지속성) 트랜잭션 지원
  • SQL을 사용한 데이터 조회 및 조작

5. SQL의 기본 구성 요소

SQL은 크게 세 가지 유형의 언어로 구성되어 있습니다.

5.1 DDL (Data Definition Language)

데이터베이스 구조를 정의하고 변경하는 명령어

CREATE - 데이터베이스 객체 생성
ALTER - 기존 객체 구조 변경
DROP - 객체 삭제
TRUNCATE - 테이블의 모든 행 삭제

5.2 DML (Data Manipulation Language)

데이터를 조작하는 명령어

SELECT - 데이터 조회
INSERT - 데이터 추가
UPDATE - 데이터 수정
DELETE - 데이터 삭제

5.3 DCL (Data Control Language)

데이터베이스 접근 권한을 제어하는 명령어

GRANT - 사용자 권한 부여
REVOKE - 사용자 권한 회수
